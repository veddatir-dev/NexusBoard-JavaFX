# Quick Reference Card - Build Fat JAR

## In 3 Steps

### STEP 1: Clean (One-time)
```powershell
Remove-Item src/main/java/com/nexus/model/*.class -Force
```

### STEP 2: Build
```bash
mvn clean package -DskipTests
```

### STEP 3: Run
```bash
java -jar target/NexusBoard-3.0.0-executable.jar
```

## FAQs

**Q: What about .class files?**  
A: They'll be regenerated by Maven. Safe to delete from `src/main/java/com/nexus/model/`

**Q: Why 152 MB?**  
A: Fat JAR includes entire JavaFX 21 frameworks (controls, graphics, fxml, swing)

**Q: Can I distribute just this one JAR?**  
A: Yes! No dependencies needed. Just `java -jar NexusBoard-3.0.0-executable.jar`

**Q: Do I need Maven installed?**  
A: Yes, or use `mvnw` wrapper if available

**Q: Why does build take 50 seconds?**  
A: Shade plugin bundles 152 MB of dependencies. Normal for first build.

**Q: Can I reduce JAR size?**  
A: Set `<minimizeJar>true</minimizeJar>` in pom.xml, but not recommended

**Q: How do I verify the build worked?**  
A: Check `target/NexusBoard-3.0.0-executable.jar` exists, then run it

## Issues & Solutions

| Issue | Solution |
|-------|----------|
| Build fails with "unknown artifact" | Run `mvn dependency:resolve` first |
| JAR won't run | Ensure you use the `-executable.jar` version |
| Out of memory | Add `-Xmx2G` before jar: `java -Xmx2G -jar ...` |
| Maven not found | Install Maven or use `mvnw` wrapper |
| Multiple JARs created | Rebuild with Maven 3.6+ for Shade plugin |
| .class files still showing | Use `-Force` flag: `Remove-Item ... -Force` |

## Pro Tips

💡 **Skip tests for faster build:**
```bash
mvn clean package -DskipTests
```

💡 **Run with more memory:**
```bash
java -Xmx4G -jar target/NexusBoard-3.0.0-executable.jar
```

💡 **Show build progress:**
```bash
mvn clean package -X  # (verbose mode)
```

💡 **Create launcher script** (Windows):
```batch

java -Xmx2G -jar "%~dp0NexusBoard-3.0.0-executable.jar"
```

💡 **Create launcher script** (Linux/Mac):
```bash

```

## Build Lifecycle

```
mvn clean               → Remove target/ folder
mvn compile             → Compile .java to .class
mvn test                → Run unit tests
mvn package             → Create JARs (shade plugin runs)
mvn install             → Install to local Maven repo
mvn deploy              → Upload to remote repo

mvn clean package       → Recommended (clean + compile + test + package)
```
